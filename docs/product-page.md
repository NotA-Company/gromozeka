## Features that actually ship

### Multi-Provider AI, No Single Point of Failure

Plug in OpenRouter, OpenAI, YandexCloud, or spin up your own OpenAI-compatible endpoint. Gromozeka routes to the model you pick and **auto-fails over** if the primary goes down. You don't lift a finger — your users never see an error.

```toml
[[ai.models]]
group = "gpt4"
apiType = "openai"
modelId = "gpt-4o"
```

### AI That Does Things, Not Just Talks

- **Function calling** — let the AI use tools and APIs on your behalf
- **Structured output** — force JSON schemas so you get usable data, not rambling
- **Image generation** — DALL-E, Kandinsky, whatever model you wire in
- **Image analysis** — send a sticker, get a explanation. Send a photo, get a description

### ML-Powered Spam Detection

A Naive Bayes classifier that **learns from your chats**. Mark something as spam (`/spam`), mark false positives as ham (`/learn_ham`), and watch it get smarter. No external service, no data leaving your infrastructure.

### Divination That's Actually Fun

Tarot, runes, and LLM-discovered layouts — not just "here's a card," but rich, contextual readings generated by the same AI that powers everything else.

### Real-World Integrations Out of the Box

| Integration | What it does |
|---|---|
| **Weather** (OpenWeatherMap) | Forecasts with geocoding — ask "What's the weather in Prague?" |
| **Web Search** (Yandex Search) | Live search with caching and rate limits |
| **Chat summarization** | Summarize long conversations into bite-sized recaps |
| **File storage** (S3/local) | File storage that scales with you |

### Built for Scale, Not Just a Script

```
main.py
  ├── TelegramBotApplication   →  hooks into python-telegram-bot
  ├── MaxBotApplication         →  custom Max Messenger client
  └── HandlersManager           →  15+ handlers, ordered pipeline, parallel or sequential
        ├── MessagePreprocessorHandler  (saves media)
        ├── SpamHandler                 (filters garbage)
        ├── CommonHandler               (standard commands)
        ├── LLMMessageHandler           (AI catch-all — always last)
        └── ... plus weather, search, divination, and more
```

Every handler is isolated. Every service is a singleton. Every message flows through the same pipeline regardless of platform. Add a new handler without touching existing code.

## Getting started in 60 seconds

```bash
git clone <repo-url> && cd gromozeka
make install
# drop your bot token and API keys into .env
./run.sh
```

Boom. Your bot is online. Telegram, Max, or both. With AI, weather, search, spam detection, and divination — all from the first command.
